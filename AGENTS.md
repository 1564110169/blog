# AGENTS.md

## 🎯 项目目标
这是一个基于 Astro 的个人博客项目，支持本地前台 + 后台（admin）运行。

本项目优先保证：
- 结构清晰
- 可维护性
- 启动简单

---

## 📁 目录约定

- src/ → 前端页面与组件（不要随意改结构）
- public/ → 静态资源 & admin 面板
- scripts/ → Node 启动脚本（核心逻辑）
- tools/ → 启动脚本（.cmd）
- logs/ → 所有日志文件

---

## 🚫 禁止修改（非常重要）

除非用户明确要求，否则：

- 不要修改 src/ 下的 UI 和页面逻辑
- 不要改动 public/ 中的 admin 结构
- 不要随意重构项目结构
- 不要引入新的框架或依赖

---

## ⚙️ 启动规则

项目启动方式：

- 前台：tools/start-frontend.cmd
- 后台：admin：tools/start-admin.cmd
- 全部：tools/start-all.cmd

修改时必须保证：
- 所有脚本可以从项目根目录执行
- 路径必须正确（tools 在子目录）

---

## 🧹 代码修改原则

- 优先最小修改（minimal change）
- 不要顺手优化无关代码
- 不要删除文件，除非明确要求
- 修改前先分析影响范围

---

## 📦 构建规则

每次修改后：

必须执行：

npm run build

如果失败：
- 只修复当前修改引起的问题
- 不做额外优化或重构

---

## 📝 日志规则

- 所有日志必须写入 logs/ 目录
- 不允许写到项目根目录

---

## 📌 输出要求

完成任务后必须说明：

- 修改了哪些文件
- 是否构建成功
- 是否存在潜在风险

---

## 🧠 行为原则

- 优先保证稳定性 > 新功能
- 优先保证结构清晰 > 代码花哨
- 不做“自作聪明”的优化
